# Contributing Guidelines

Thank you for your interest in contributing to the OSINT Dashboard project!

## Ethical Standards
- This project is strictly for **educational and professional OSINT purposes**.
- It must never be used to access or store illegal content such as CSAM.
- Contributions must respect legal and humanitarian boundaries.

## How to Contribute
1. Fork the repository.
2. Create a new branch for your feature or fix.
3. Make your changes with clear documentation.
4. Submit a pull request with a descriptive message.

## Code Style
- Use Python best practices (PEP8).
- Keep functions modular and documented.
- Ensure reproducibility with updated `requirements.txt`.

## Reporting Issues
- Use the GitHub Issues tab.
- Provide clear steps to reproduce the problem.
- Suggest possible fixes if you can.

---

By contributing, you agree to uphold the ethical standards outlined above.
